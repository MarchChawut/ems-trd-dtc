## 2. ⚠️ ฟิลด์/ข้อมูลที่ยังนำเข้าแบบมีโครงสร้างไม่ได้ (ต้องพิจารณาปรับปรุงเว็บ)

### 2.1 ชื่อผู้ครอบครองที่ไม่ตรงกับพนักงานในระบบ (12 ชื่อ)
schema มีเพียง `currentHolderId` (FK → `users`) ไม่มีฟิลด์ข้อความสำหรับชื่อผู้ครอบครอง
จึงเก็บชั่วคราวไว้ใน `notes` ("ผู้ครอบครอง: ...") — ค้นหา/กรองตามผู้ครอบครองไม่ได้

แยกได้ 2 กลุ่ม (ตรวจกับตาราง `users` แล้ว):

**ก. พิมพ์ชื่อต่างกัน — มีบัญชีในระบบ แค่สะกดไม่ตรง → แก้สะกดให้ตรงแล้วรันซ้ำจะ match อัตโนมัติ**
- `น.ส.จุฬารัชต์ มากเปรมบุย` ← ระบบมี "จุฬารัชต์ มากเปรม**บุญ**"
- `นายฐากูร ตั้งชนะชัยพง**ศ์**` ← ระบบมี "ฐากูร ตั้งชนะชัยพง**ษ์**"
- `นายณรงค์ฤทธิ์ ศรีนวล **iMac` (มี annotation ต่อท้าย)

**ข. ไม่มีบัญชีในตาราง `users` → ต้องสร้าง User หรือใช้ field ข้อความ**
`นายณรงค์ฤทธิ์ ศรีนวล`, `น.ส.วารีรัตน์ มากมี`, `น.ส.วาสิฎฐี มุสิกะพันธ์`,
`นาย วิชาญ วชิรภูบาลภักดี`, `ร.อ.รพีธรรม วงศ์สกุลคำโคตร`, `พ.อ.ภูษิต คล้ายหิรัญ`,
`นายพชร เอี่ยมสุรีย์`, `นาย คมศร ทองบุญมา`, `พล.อ.อ.อำนาจ จีระมณัมัย`

**ข้อเสนอ:** (ก) แก้การสะกด/สร้าง `User` ที่ขาด แล้วรันสคริปต์ซ้ำ (idempotent)
หรือ (ข) เพิ่ม field `holderName` (ข้อความ) ใน `Asset` รองรับผู้ครอบครองที่ไม่มีบัญชี (เช่น บุคคลนอกกอง)

### 2.2 จำนวน + หน่วย (Quantity/Unit)
`Asset` มองครุภัณฑ์เป็น **1 ชิ้น/แถว** ไม่มีฟิลด์ `quantity`/`unit`
- แผ่น "อุปกรณ์ต่อพ่วง" มีจำนวน เช่น "24 ชิ้น", "2 เส้น" → เก็บใน `notes` ("จำนวน: ...") เท่านั้น
- ตารางสรุปยอดรวมรายหมวด (คอลัมน์ I/J) → **ไม่ได้นำเข้า** (เป็นข้อมูลสรุป ไม่ใช่รายชิ้น)

**ข้อเสนอ:** ถ้าต้องนับครุภัณฑ์แบบกอง/วัสดุสิ้นเปลือง เพิ่ม `quantity` + `unit`
หรือพิจารณาย้ายอุปกรณ์ต่อพ่วงไปโมดูล "วัสดุ" (Supply) ที่มีฟิลด์จำนวนอยู่แล้ว

### 2.3 assetTag ว่างสำหรับเลขครุภัณฑ์ซ้ำ (25 แถว / 12 เลข)
มีเลขครุภัณฑ์ซ้ำกัน 12 เลข (เช่น `744000968000266` ใช้กับเครื่องพิมพ์ 3D 3 เครื่อง)
เนื่องจาก `assetTag` มี unique constraint จึงเว้น `assetTag=null` แต่ยังเก็บค่าเต็มใน `serialNumber` (ค้นหาได้)

**ข้อเสนอ:** ตรวจสอบเลขครุภัณฑ์ซ้ำกับต้นทาง — ถ้าซ้ำจริงควรออกเลขย่อย หรือยกเลิก unique บน `assetTag`

### 2.4 ยี่ห้อ/รุ่น (brand/model) ยังไม่แยกฟิลด์
คอลัมน์ "คุณสมบัติ/รายละเอียด" เก็บเป็นข้อความรวมใน `notes` (เช่น "ยี่ห้อ Lucky รุ่น D3-75120")
ฟิลด์ `brand`/`model` ที่มีอยู่ใน schema **ยังว่าง**

**ข้อเสนอ:** ทำ parser แยก ยี่ห้อ/รุ่น ภายหลังถ้าต้องการกรอง/รายงานตามยี่ห้อ

### 2.5 ฟิลด์ที่ schema มีแต่ไฟล์ไม่มีข้อมูล
`acquisitionDate`, `acquisitionCost`, `documentNumber`, `condition`, `receiverName`,
`lastInspectionDate/Condition/By` → ใช้ค่า default (`condition=GOOD`) เพราะไฟล์ไม่มีข้อมูลส่วนนี้

## 3. หมายเหตุ / สิ่งที่ควรดำเนินการต่อ

1. **ข้อมูลทดสอบเดิม** — มี asset `id=1` (`name="gdgdhj"`) เป็นข้อมูลขยะทดสอบ
   อยู่นอกหมวด import (ไม่ถูกแตะ) → แนะนำให้ลบทิ้ง
2. **PII / Routing** — ไฟล์มีชื่อพนักงานจริง และ CLAUDE.md ระบุ TRD ห้ามประมวลผลบน cloud
   งานนี้ทำบน cloud model ตามที่ผู้ใช้ยืนยัน — ควรรับทราบไว้
3. รันซ้ำได้ปลอดภัย: สคริปต์ลบ asset ในหมวดของ import ก่อนเขียนใหม่ทุกครั้ง (idempotent)
